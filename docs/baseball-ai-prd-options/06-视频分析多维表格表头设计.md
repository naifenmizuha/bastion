# 06 视频分析多维表格表头设计

## 1. 表定位

表名建议：`DrillCard`

这里的视频分析表就是训练卡片表 `DrillCard`，不再单独拆 `VideoAnalysis`、`TrainingSourceInbox` 或其他中间表。

MVP 阶段只做一件事：把一个训练视频链接和提交者的推荐理由存进表里，由 AI 生成摘要，供教练后续判断是否采用。

## 2. 极简表头

| 中文表头 | 字段 key | 飞书字段类型 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| 提交人 | `submitted_by` | 人员 | 是 | 谁提交的视频 |
| 提交时间 | `submitted_at` | 日期时间 | 是 | 提交记录的时间 |
| 视频链接 | `video_url` | URL | 是 | 训练视频原始链接 |
| 推荐原因 | `recommendation_reason` | 长文本 | 是 | 提交人为什么认为这个视频值得收集 |
| 适用位置 | `target_positions` | 多选 | 否 | 投手 / 捕手 / 内野 / 外野 / 跑垒 / 打击等 |
| 人数限制 | `participant_requirement` | 数字 | 是 | 该训练最少人数 |
| 场地限制 | `venue_requirement` | 单选 | 是 | 无限制 / 室内场地 / 有网小场 / 完整场地 |
| AI 摘要 | `ai_summary` | 长文本 | 否 | AI 对视频训练内容的简要总结 |
| 是否启用 | `is_enabled` | 勾选框 | 是 | 默认为否，只有教练有权限修改 |


## 3. 字段说明

### 3.1 提交人

用于追踪来源。后续如果教练觉得某个队员或助教经常提交高质量内容，可以反向鼓励。

### 3.2 提交时间

用于排序和追踪最近新增训练素材。飞书可设置为自动填入创建时间。

### 3.3 视频链接

保存原始视频链接即可。MVP 阶段不做复杂 URL 标准化、去重或平台字段拆分。

### 3.4 推荐原因

这是最重要的人工上下文。AI 只看视频标题或简介时可能不知道队伍为什么需要它，推荐原因可以告诉 AI 和教练：

- 这个视频解决什么问题。
- 为什么适合本队。
- 想给哪个位置或哪类球员看。
- 希望后续训练方案如何使用。

### 3.5 适用位置

建议多选项：

- 投手
- 捕手
- 内野
- 外野
- 一垒
- 二垒
- 三垒
- 游击
- 跑垒
- 打者
- 全队

### 3.6 AI 摘要

AI 摘要只做简短提炼，不在这个表里展开完整训练卡片结构。

推荐格式：

```text
视频内容：
适合对象：
关键训练点：
教练可关注：
```

## 4. 推荐视图

### 4.1 按提交时间

默认视图，最新提交排在最上面。

显示字段：

- 提交时间
- 提交人
- 视频链接
- 推荐原因
- 适用位置
- AI 摘要

### 4.2 按适用位置

按 `适用位置` 分组，方便教练快速找某类训练素材。

### 4.3 待补 AI 摘要

过滤条件：

- `AI 摘要` 为空。

用途：

- Agent 定期扫描这个视图，为新视频补充摘要。

## 5. Agent 处理规则

1. Agent 只读取 `AI 摘要` 为空的记录。
2. Agent 使用 `视频链接`、`推荐原因`、`适用位置` 生成摘要。
3. Agent 只写入 `AI 摘要`。
4. Agent 不修改 `提交人`、`提交时间`、`视频链接`、`推荐原因`、`适用位置`。
5. 如果视频链接无法访问，AI 摘要写明“无法自动读取视频内容，仅根据推荐原因生成初步摘要”。
